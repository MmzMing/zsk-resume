# 前端开发规范 (Frontend Development Standards)

## 1. 项目概述

本项目是一个基于 React + TypeScript + Vite 的在线简历编辑器，使用 Zustand 进行状态管理，Tailwind CSS 进行样式设计，并集成 Tiptap 富文本编辑器。

### 技术栈

| 类别 | 技术 |
|------|------|
| 框架 | React 19 + TypeScript |
| 构建工具 | Vite 8 |
| 状态管理 | Zustand |
| 样式方案 | Tailwind CSS 4 + shadcn/ui |
| 富文本编辑 | Tiptap |
| PDF 生成 | @react-pdf/renderer |
| 拖拽排序 | @dnd-kit |
| 代码规范 | ESLint + TypeScript-ESLint |
| 部署 | Cloudflare Workers (Wrangler) |

## 4. React 组件规范

### 4.1 组件定义

- 使用函数组件，不使用类组件
- 组件名称使用 PascalCase
- 导出使用命名导出，避免默认导出

```typescript
// ✅ 推荐
export function ResumePreview({ data }: ResumePreviewProps) {
  return <div>{data.name}</div>;
}

// ❌ 避免
const resumePreview = (props) => { ... }
export default ResumePreview;
```

### 4.2 Hooks 使用规范

- Hooks 必须在组件顶层调用，不能在条件语句或循环中调用
- 自定义 Hook 以 `use` 开头
- 依赖数组必须完整声明所有依赖项

```typescript
// ✅ 自定义 Hook
export function useMediaQuery(query: string): boolean {
  const [matches, setMatches] = useState(false);

  useEffect(() => {
    const media = window.matchMedia(query);
    setMatches(media.matches);
    
    const listener = (e: MediaQueryListEvent) => setMatches(e.matches);
    media.addEventListener('change', listener);
    
    return () => media.removeEventListener('change', listener);
  }, [query]);

  return matches;
}
```

### 4.3 性能优化

- 使用 `React.memo` 优化纯组件渲染
- 使用 `useMemo` 缓存计算结果
- 使用 `useCallback` 缓存函数引用

```typescript
// ✅ 组件级优化
export const ResumeCard = React.memo<ResumeCardProps>(({ resume }) => {
  return <div>{resume.name}</div>;
});

// ✅ 计算缓存
const filteredSections = useMemo(() => 
  sections.filter(s => s.visible),
  [sections]
);

// ✅ 函数缓存
const handleClick = useCallback((id: string) => {
  setSelected(id);
}, []);
```

## 5. 状态管理规范 (Zustand)

### 5.1 Store 定义

```typescript
// ✅ Store 命名：XxxStore.ts
// ✅ 使用 immer 中间件处理复杂状态更新
import { create } from 'zustand';

interface ResumeState {
  resumeData: ResumeData;
  selectedSection: string | null;
  updateResumeData: (data: Partial<ResumeData>) => void;
  setSelectedSection: (id: string | null) => void;
}

export const useResumeStore = create<ResumeState>((set) => ({
  resumeData: defaultResumeData,
  selectedSection: null,
  updateResumeData: (data) =>
    set((state) => ({
      resumeData: { ...state.resumeData, ...data },
    })),
  setSelectedSection: (id) => set({ selectedSection: id }),
}));
```

### 5.2 Store 使用规范

- 在组件中直接调用 Hook，不需要 Provider 包裹
- 使用选择器避免不必要的重渲染
- 复杂状态更新使用函数式更新

```typescript
// ✅ 使用选择器
const resumeData = useResumeStore((state) => state.resumeData);
const updateResumeData = useResumeStore((state) => state.updateResumeData);

// ❌ 避免：订阅整个 store 导致不必要的重渲染
const { resumeData, updateResumeData } = useResumeStore();
```

## 6. 样式规范 (Tailwind CSS)

### 6.1 类名组织

- 按照布局、尺寸、间距、颜色、交互的顺序组织类名
- 使用 `cn()` 工具函数合并类名

```typescript
// ✅ 推荐的类名顺序
<div className="
  flex items-center justify-between  // 布局
  w-full h-12 px-4 py-2             // 尺寸
  gap-2 mb-4                        // 间距
  bg-white text-gray-900            // 颜色
  hover:bg-gray-100 transition      // 交互
">
  Content
</div>
```

### 6.2 自定义工具函数

```typescript
// src/lib/utils.ts
import { clsx, type ClassValue } from 'clsx';
import { twMerge } from 'tailwind-merge';

export function cn(...inputs: ClassValue[]) {
  return twMerge(clsx(inputs));
}
```

### 6.3 响应式设计

- 使用移动优先的断点系统
- 断点：`sm:` (640px), `md:` (768px), `lg:` (1024px), `xl:` (1280px)

```typescript
// ✅ 移动优先
<div className="
  w-full 
  md:w-1/2 
  lg:w-1/3 
  xl:w-1/4
">
```

## 7. 组件开发规范

### 7.1 组件拆分原则

- 单一职责：每个组件只负责一个功能
- 复用性：超过 50 行考虑拆分
- 可测试性：每个组件应易于单元测试

### 7.2 基础 UI 组件

- 使用 shadcn/ui 作为基础组件库
- 不要修改 `components/ui/` 下的文件（这些是自动生成/管理的）
- 通过 props 自定义组件行为

### 7.3 业务组件

- 放在对应的业务目录下 (如 `components/editor/`, `components/preview/`)
- 组件文件名使用 PascalCase
- 组件导出不使用 barrel export（避免打包体积增大）
